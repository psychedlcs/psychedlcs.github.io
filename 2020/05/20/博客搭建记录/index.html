博客搭建记录

用GitHub+Hexo搭建博客的一个流程记录。

1. 建站原因

过去两年里我由于专业&兴趣爱好&工作需求码了很多字，并察觉到我非常喜欢回看自己写的东西，但是似乎没有做过任何像样的学习笔记（除了高中数学和Fundamentals of Phonology，因为老师要检查……真是感谢两位老师……）

最近刚好回坑ron2需要写点笔记，加上马上要去读书，遂决定建立一个站点培养笔记习惯。

可能会有的内容

  • 麻雀（ron2复盘，牌谱学习，观战记，杂谈，etc.）
  • 专业相关 (marketing, communication, second language acquisition, pragmatics, etc.)
  • 课堂笔记
  • 翻译（书籍&文章）

需要的功能

  • 界面不能太难看（颜控的基本素质）
  • 多端同步阅读/编辑
  • 分版块加密（因为可能会上传有版权限制的东西内容，退而求其次也需要分文章加密）
  • 防搜索（非必需，估计很少人搜得到）

否决了以下方案：

  • 各种本地笔记（Zine、有道云等）：多端同步阅读不便，经常容易戳到编辑按钮。另外我总觉得一个东西不发布出去就不算写完……
  • Lofter：搭建方便，子博客可加密，诸方面功能都比较合适。然而：①有被莫名炸号的经历，投诉起来非常麻烦；②自我河蟹功能完善，敏感词很多。
  • WordPress：加载慢。不优化的样式实在太难看了……优化起来又非常麻烦，那个模块编辑器难用到窒息。
  • Alice+：为数不多可以实现分版块加密的网站平台，建设也比较方便。但是广告太多。而且我总觉得它下一秒就要倒闭的样子……

2. 目前执行方式

GitHub + Hexo：样式还OK。搭建过程没有想象中的麻烦。大概也许没有分版块加密功能，不过可以实现文章加密，姑且应急一用。

Markdown

目前使用有道云笔记。优势如下：

  • 可多端同步编辑。Windows端可直接导出md文件。
  • 可自动实现备份功能。同时撰写文章可以自动保存，不太需要担心数据丢失。
  • 可直接作为图床使用。假设这个功能哪天被端了，也大概可以通过简单的氪金来解锁。
  • 颜值高。Markdown编辑器半可视化，甚至还有主题可以选。

辅助程序：Typora，用于Hexo相关md文件的修改和优化。

域名

现在买域名似乎都要备案，除了更有排面以外似乎也没什么实际意义，遂暂时不买。不过随时可以搞一个就是了。

3. 残留问题

  • 系统同步：不远的将来很可能要换电脑，加上近期很可能会使用Mac OS进行编辑，需要研究一下本地内容如何同步。应该不难就是了。
  • 防搜索：防止主流搜索引擎抓取。印象中可以通过比较简单的代码实现。我感觉大部分程序员老哥们都迫不及待地想把自己的博客推广出去……
  • 存续情况：暂且不清楚GitHub托管对容量/时长等方面的限制。需要了解一下。不过无论如何Hexo还是比较好用的，如果今后有需求会考虑虚拟机/域名等等。

End.
